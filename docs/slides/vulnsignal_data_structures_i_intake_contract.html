Data Structures I: Source Intake Contract

VulnSignal starts with evidence-bearing source tasks, then normalizes every source into (task_instance, candidate_location) records with checker/oracle truth.
E022 Contract

Input Type A: Dynamic Oracle Corpora

  • Sanitizer crash, reproducer, fuzz test, KUnit, or syzkaller-style case.
  • Requires source snapshot, build/test command, observed behavior, and affected component.
  • Produces strongest labels when the behavior is reproducible.

Input Type B: CodeQL / Checker Tasks

  • CodeQL database plus normalized semantic facts.
  • Rule-specific PASS / FAIL / UNKNOWN from lifecycle or security checkers.
  • Conditional truth: valid for the encoded rule and available facts.

Input Type C: Patch / Advisory Sources

  • CVEfixes, OSV, MoreFixes, GHSA, or project security patches.
  • Used for candidate expansion and weak labels only by default.
  • Upgraded only when tied to CodeQL facts, before/after evidence, or an oracle.
Normalize source snapshot, evidence, and rule context
Generate candidate locations and CodeQL fact links
Assign label source and confidence without hiding UNKNOWN

Rejected As Ground Truth

  • LLM consensus alone.
  • Regex or custom parser flags alone.
  • Generic vulnerable / non-vulnerable labels alone.
  • Historical patch labels alone.
  • Human opinion without reproducible evidence or checker output.
