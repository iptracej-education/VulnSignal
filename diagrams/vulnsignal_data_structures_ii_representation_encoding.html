Data Structures II: Source Code Representation

v0 consumes three explicit views: bounded source-token sequences, typed CodeQL fact/path records, and task-level candidate lists. Graphs are derived later, not the first training format.
Representation v0

Source Sequence View

  • JSONL row keyed by candidate_id.
  • Deterministic token list from candidate file/function/line window.
  • Consumed by source encoder as one bounded sequence per candidate.

Fact / Path Record View

  • Typed records: fact ID, kind, predicate, object ID, source anchor.
  • Optional edge/path fields: src, dst, edge kind, path index.
  • Consumed by fact/path encoder; graph view is derived later.

Task Candidate List View

  • One task row points to many candidate IDs.
  • Labels live separately with strength and provenance.
  • Consumed by pairwise/listwise ranker inside each task_id.

v0 Model Decision

  • Source sequence encoder.
  • Fact/path record encoder.
  • Task-level listwise or pairwise ranking head.
  • Prediction: suspicious location, object, rule, evidence, test guidance, confidence / UNKNOWN.

Deferred, Not Discarded

  • Graph model is deferred until dataset quality justifies it.
  • Graph-ready IDs are still stored now.
  • Language-model assistance is allowed for triage and enrichment, not truth.

Not The Primary Design

  • No generic vulnerable / non-vulnerable classifier.
  • No language-model-first architecture.
  • No reward model before checker-grounded data exists.
